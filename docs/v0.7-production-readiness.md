# v0.7 Production Readiness Layer

Implemented in full by the `v1.0.0` release surface.

`v0.7.0` closes the last obvious pre-`1.0` gaps that still made MTP feel like a
strong control plane without a complete operator story.

## What v0.7 adds

- `mtp-extract`: transcript/export -> schema-valid draft package
- `mtp-benchmark`: benchmark suites, benchmark results, adapter certification artifacts
- stronger registry signing surface: `ed25519` in addition to `hmac-sha256`
- repo examples for extraction and benchmark artifacts

## Why this matters

Before `v0.7`, MTP covered:

`Validate -> Execute -> Report -> Compare -> Version -> Trust`

but `Extract` was still mostly a prompt template, and `community benchmarks`
were still roadmap text. `v0.7` makes both of those executable.

## What still belongs to v1.0

- KMS-backed or enterprise key-management integrations
- published provider certification matrix across real platforms
- compatibility/support contract for long-lived production deployments
- enterprise reference architecture and governance patterns
